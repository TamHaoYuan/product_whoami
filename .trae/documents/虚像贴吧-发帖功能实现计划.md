# 虚像贴吧 - 发帖功能实现计划

## 概述
为虚像贴吧添加发帖功能，用户无需注册账户，可以自定义发帖名称、头像，可以选择身份（包括普通用户、版主、管理员、站长），管理员和站长身份需要密码认证。

## 目标
- 添加发帖功能，用户可以发布新帖子
- 支持自定义用户名和头像
- 支持选择身份（普通用户、版主、管理员、站长）
- 管理员和站长身份需要密码验证
- 帖子数据存储在客户端（localStorage）
- 保持现有功能不变

## 技术方案

### 数据存储
- 使用 localStorage 存储帖子数据
- 帖子数据结构：
  ```typescript
  interface Post {
    id: string;
    title: string;
    content: string;
    author: {
      name: string;
      avatar: string;
      role: 'user' | 'moderator' | 'admin' | 'owner';
    };
    pubDate: Date;
    tags: string[];
    category: string;
  }
  ```

### 身份系统
- 身份选项：普通用户、版主、管理员、站长
- 管理员/站长需要密码验证才能选择
- 密码存储在代码中（简单验证）或环境变量中

## 验收标准

### AC-1: 发帖功能
- **给定**：用户访问发帖页面
- **当**：用户填写帖子标题、内容、选择身份后点击发布
- **Then**：帖子成功发布并显示在帖子列表中
- **验证**：`programmatic`

### AC-2: 身份选择
- **给定**：用户在发帖表单中选择身份
- **当**：用户选择普通用户或版主时
- **Then**：可以直接选择，无需验证
- **验证**：`programmatic`

### AC-3: 管理员认证
- **给定**：用户选择管理员或站长身份
- **当**：用户点击发布按钮
- **Then**：弹出密码验证框，验证成功后才能发布
- **验证**：`programmatic`

### AC-4: 帖子显示
- **Given**: 用户访问网站首页
- **当**：网站加载时
- **Then**：显示本地存储的帖子列表
- **验证**：`human-judgment`

### AC-5: 身份标识
- **Given**: 用户查看帖子
- **当**：帖子作者选择了一定的身份
- **Then**：在作者名称旁边显示对应的身份标识
- **验证**：`human-judgment`
